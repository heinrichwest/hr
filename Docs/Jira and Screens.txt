# A) Jira-ready backlog (Epics → Stories → Subtasks)

## Jira standards (apply to all stories)

**Story format:**
“As a `<role>`, I want `<capability>`, so that `<benefit>`.”

**Definition of Done (DoD):**

* UI + API complete
* Validation rules implemented
* RBAC permissions enforced
* Audit logs for create/update/delete actions
* Tests (unit + integration where relevant)
* Error handling + user messages
* Documentation updated (admin + user help notes)

---

## EPIC 0 — Discovery & Specification

**Goal:** Finalise policy rules, outputs, exports, and acceptance tests.

**Stories**

* **0.1 Requirements workshops & policy capture**

  * Subtasks: workshop sessions (HR/Payroll/IR/Finance), confirm leave policies, confirm payroll elements, confirm IR templates & warning durations, confirm approval chains.
* **0.2 Produce PRD + wireframes**

  * Subtasks: module wireframes, screen inventory, user journeys, edge-case list.
* **0.3 Create data dictionary**

  * Subtasks: entities, fields, relationships, retention rules.
* **0.4 Define outputs & export specifications**

  * Subtasks: bank file formats, GL export mapping, statutory dataset requirements, report list.
* **0.5 UAT plan & test scripts**

  * Subtasks: role-based scripts, payroll reconciliation tests, IR case tests.

**Acceptance criteria**

* Signed-off PRD, wireframes, roles matrix, policies, and UAT scripts.

# C) Import templates (minimum required)

1. **Employees import**

* employee_no, first_name, last_name, id/passport, dob, start_date, department, branch, manager, contract_type, salary_type, salary, pay_frequency, tax_ref, UIF_flag, bank fields…

3. **Pay elements import**

* element_name, type, calc_method, defaults, flags, GL code

4. **YTD payroll import (for mid-year go-live)**

* employee_no, YTD taxable income, YTD PAYE, YTD UIF, YTD SDL, etc.
